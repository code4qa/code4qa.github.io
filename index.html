Creating QA Web Page Test Automation Processes Using Java, Selenium, TestNG, Log4J, Properties and JXL

For the Initial automation of the testing of web sites, I use the following techniques:

  • Create a Page Objects Model of Java Classes for each web page and Methods for each field, and Classes of Actions.
  • Use the Selenium Web Driver to navigate through the web site using the Page Object Model.
  • Drive the automated tests using TestNG for tests in sequence or randomly with automatic report generation.
  • Keep detailed logs in files and summary logs shown on the console using Log4J.
  • Configure the tests for multiple environments using Java Properties files.

For more advanced features:

  • Write test results to Excel Spreadsheets using the Java Library JXL, create the spreadsheets too.
  • Read test data from Excel Spreadsheets to drive the TestNG tests using JXL.
  • Automatically populate test data in test actions from properties using Java Introspection.
  • Generate emails with attachments of screen shots, TestNG reports and log files.
